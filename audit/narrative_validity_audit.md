# 敘事流程效度審計 —《敲牆的人》ch01

> 借用 validity-audit 兩段式。把量化的「洩漏/算術 bug/協定 bug」對應成敘事的
> 「斷鏈 / 接點缺失 / 狀態-劇情不一致」。**否證的假設:「這一章的劇情流程是連貫的」。**
> 對象檔:`src/data/chapters/ch01_knocking.json`、`src/engine/reducer.ts`、`src/ui/*`。

## 受查宣稱（headline）
1. 玩家的每個選項都接得上後續故事。
2. 白天→夜晚、夜晚各層之間的接點通順。
3. 整體流程連貫、不混亂。

---

## 第 1 段:內部機械審計

### 🔴 CRITICAL — F1. 抉擇不分支:被「判死」的選項仍照跑後續所有層
- **機制**:`reducer.ts` 的夜晚是**循序闖關**(nightChoiceIndex 0→4 全部都要過),結局在**最後**才用旗標/數值結算。選項的 `outcome` **不影響下一層的 prompt**。
- **後果(對應使用者抱怨「有些選項的故事沒有接在一起」)**:
  - L1 選**開門查看** → outcome 是恐怖高潮(門後是她中風倒地的殘影、「她猛地回過頭」),且 `door_opened` 已**鎖定 worst 結局**。但玩家**仍被帶去 L2 收音機 → L3 哭聲 → L4「你是他嗎」→ L5 送別**,彷彿什麼都沒發生、還能平靜地替她哼歌、補三下。**動作與後續完全脫節。**
  - L1 **不理會**(牆面裂開、她更激動)→ 下一秒 L2 prompt 卻是平靜的「收音機自己響了」,情緒不接。
- **這是「流程混亂」與「故事不接」的最大單一來源。**

### 🔴 CRITICAL — F2. 白天→夜晚沒有過場(使用者明指)
- **機制**:`reducer.ts` 最後一個白天互動 `ADVANCE_DIALOGUE` 完 → 直接 `phase:"night"`、`nightChoiceIndex:0`。中間**沒有任何敘述拍**。
- **後果**:白天最後一拍是手勢互動(「指尖一下、一下地點」)+ hint「天，慢慢黑了」,**下一拍**直接是夜晚 L1「**後半夜**，牆的另一頭響起敲擊」。從黃昏到後半夜的時間跳躍、白天溫情層→夜晚恐怖層的調性切換,**完全沒有橋接**(沒有「翻開守則」「燈滅了」「陰時降臨」這類過場)。VN 體感是硬切。

### 🟠 MAJOR — F3. 夜晚層之間缺連接組織(outcome→下一 prompt 不咬合)
- L1 outcome（回敲後「敲擊又繼續，不對，還不對」）→ L2 prompt 直接「深夜，收音機自己響了」,兩件事沒有因果或時間連接詞。L2→L3、…多處同樣是「各自獨立的場景卡」並排,而非一條連貫的夜。

### 🟡 MINOR — F4. 夜晚時間標記順序矛盾
- L1 prompt =「**後半夜**」,L2 prompt =「**深夜**」。深夜(約 23–1 點)在時間上**早於**後半夜(約 2–4 點)。先後半夜再深夜 = 時間倒退。L3/L4/L5 則靠「哭聲歇了」「天就要亮了」往前推,唯 L1↔L2 逆序。

### 🟡 MINOR — F5. 結局文本與實際歷程可能不符
- worst(開門)結局文本只講「你沒有回敲，反而扭開了門」,但玩家在 L2–L5 可能做了聽完/跟著哼/敲三下等「對」的事;結局**無視**這些,雖由 door_opened 主導,仍是歷程與收束的輕微不一致。

### ✅ 通過項
- 旗標指涉完整性、effects/requires/when 的 var/flag 解析:`loadChapter` 三道把關 + 19 測試全綠,**無斷鏈**(機械層面 id 都接得上)。
- `learned_song` 等線性白天必蒐齊,`requires` 過濾正確(`跟著哼`/`敲三下` 條件一致)。
- 四級結局皆可達(引擎實算)。
- 開場 opening→intro→day 接點有(交接→住戶卡→白天)。

## 內部審計小結（待獨立 reviewer 對抗複查）
- **headline 1（選項都接得上）：不可信** → F1。
- **headline 2（接點通順）：不可信** → F2、F3、F4。
- **headline 3（流程連貫）：不可信** → 由 F1 主導。
- **單一最重要修正(暫定)**:F1 —— 讓「判死/重大錯誤」的選項在當層就收束(分支或提早結束夜晚),而不是讓玩家在已注定的結局上繼續空轉;其次補 F2 的日→夜過場。
- 待 Stage 2 獨立 reviewer 挑戰是否還有更深的斷裂(例如角色視角/旁白人稱、立繪表情與內容不符、選項措辭暗示結果等)。

## 第 3 段:逐項驗證與更正(誠實,未照單全收)
- **N1 立繪表情背離劇情** — 邏輯確認(rage 驅動,開門後 broken 卡到溫情高潮)。**成立 / CRITICAL。**
- **N2 開門 outcome 提前劇透真相 + 三重重播** — 讀檔確認(L1 開門 outcome ≈ truth ≈ abandoned)。**成立 / CRITICAL。**
- **N3 good(rest)結局說「你沒接到歌」但玩家可能接了** — 驗:`good.when` 不含 `soothed`;soothed+5旗標但缺 bridged → 落 rest → 文本說反話。**成立 / MAJOR。**
- **N4 opening「未必是同一個」誤導 + L5 替玩家宣告頓悟** — 讀檔確認。**成立 / MAJOR(措辭)。**
- **N5 L5 prompt/L3 label 直接報答案** — 讀檔確認(L5 prompt 公布三下)。**成立 / MAJOR。**
- **N6 door 鎖 worst 但陣列優先序可能先命中 best** — **驗後降級**:`truth_clue_knock` 需 `answered`(L1 回敲),與開門互斥 → 開門路徑永遠拿不到 truth 旗標 → **「門+best」實際不可達**。屬潛在脆弱,非現行 bug。修 F1 時順帶根治。
- **N7 線性必播放大白天台詞矛盾** — 驗:`photo`「他回了我三下」vs `gesture/fragC`「想不起幾下」並置。**成立 / MINOR(但破設定)。**
- **N8 入夜→後半夜跳空 + F4 逆序** — 確認。**成立 / MINOR。**

### 最終裁決:三個 headline 全部撤回(劇情非連貫)。
### 修正優先序(供實作)
1. **F1+N2+N6 一起修**:開門查看 → 當層短路進 abandoned 結局(跳過 L2–L5/truth);其他「重大錯誤」選項亦考慮提早收束 → 不再空跑、不再提前劇透/重播、根治陣列誤判。
2. **F2/F3/F4/N8 日→夜與夜內過場**:加入夜的橋接敘述拍 + 修時間標記順序。
3. **N1 立繪表情改由節點驅動**(非純 rage):選項/真相/結局帶表情提示。
4. **N3 結局文本**:rest 不可寫死「沒接到歌」(依 soothed 動態,或拆級)。
5. **N5 去破梗**:L5 prompt 不直接公布三下;L3 label 改中性。
6. **N7 白天台詞**:photo 與 gesture/fragC 的「記不記得三下」統一(photo 不直說三下,或 gesture 改寫)。
7. **N4 措辭**:opening「未必是同一個」改回「不是同一個面」語意;L5「你全明白了」改成不強加。
